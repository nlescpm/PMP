\textbf{2. Technology plan for “Exchange of CO2 in tropical ecosystems unravelled (EXCITED)”}}


\bigskip

\begin{itemize}[resume*=listWWNumxxxi,start=1]
\item 
\href{https://nlesc.sharepoint.com/:b:/s/all/EZ6Iu_0k89tLkdZALfb7QagBD6t2hy4UkI8jYjyMHoVFnw?e=5BldjB}{\textstyleInternetlink{EXCITED
proposal}}}
\end{itemize}

\bigskip


The goal of the EXCITED project is to create a dataset of temporally and spatially consistent CO2 emissions by natural
ecosystems (i.e., the Net Ecosystem Exchange). }


\bigskip


This dataset will be created from site-scale measurement data (Fluxnet), inverse model results (CarbonTracker), and the
$\text{\textgreek{‘}}$ERA5$\text{\textgreek{’}}$ global weather reanalysis product. }


\bigskip


Besides the resulting dataset(s), we will also provide the trained model(s) and code. }


\bigskip

\begin{enumerate}[series=listWWNumxiv,label=\arabic*.,ref=\arabic*]
\item 
\textbf{Used technologies }}
\end{enumerate}

\bigskip


The project will make extensive use of Python, due to its broad ecosystem of (netCDF) data processing and machine
learning packages. }


\bigskip


More specifically, we will make use of the following packages: }


\bigskip


\textit{Data intake/processing: }}

\begin{itemize}[series=listWWNumxiii,label=[F0B7?]]
\item 
\ \ \ \ xarray }
\item 
\ \ \ \ dask }
\item 
\ \ \ \ flox (xarray+dask extension for fast groupby operations). \ }
\end{itemize}

\bigskip


See \textcolor[HTML]{00B0F0}{https://flox.readthedocs.io/en/latest/ }}


\bigskip


With these packages, we will be able to efficiently load and process the data and use all computing cores, both on a
local PC and HPC. }


\bigskip


\bigskip


\textit{Machine learning: }}


\bigskip

\begin{itemize}[series=listWWNumxii,label=[F0B7?]]
\item 
\ \ \ \ sklearn }
\item 
\ \ \ \ pycaret (for comparing performance of different ML models) }
\item 
\ \ \ \ Possibly LightGBM (“Light Gradient Boosting Machine” a decision tree model like XGBoost, but much faster).
Mostly for performance reasons. }
\end{itemize}

\bigskip


Additionally, we will make use of ONNX to be able to save and share the trained models in \ \ an open format. }


\bigskip


Analysis and cooperation with the LA will be through Surf Research Cloud. }


\bigskip

\begin{enumerate}[resume*=listWWNumxiv,start=1]
\item 
\textbf{Technological outcomes }}
\end{enumerate}

\bigskip


In the project we will develop an efficient and reproducible workflow to generate the following outcomes: }


\bigskip

\begin{itemize}[series=listWWNumxi,label=[F0B7?]]
\item 
\ \ \ A python workflow }
\end{itemize}
\begin{itemize}[series=listWWNumx,label=[F0B7?]]
\item 
\ which preprocesses input data, trains the ML model, as well as provide some useful plotting utilities to analyze the
trained model \ }
\item 
hosted on Github, Zenodo, and the RSD. If the workflow proves to be sufficiently reusable and generic, we can publish it
as a package on pypi. }
\item 
To ensure code quality, we will make use of testing, (static) code analysis and typing (i.e., pytest, black, ruff,
mypy). Workflows will be added to Github Actions. We will guide the LA on these aspects as well, so they can maintain
the code once the project is finished. }
\end{itemize}
\begin{itemize}[series=listWWNumix,label=[F0B7?]]
\item 
\ \ \ \ The trained models in ONNX format, on Zenodo. }
\end{itemize}
\begin{itemize}[series=listWWNumviii,label=[F0B7?]]
\item 
\ \ \ \ The output datasets, where the model is applied on (global) ERA5 data. To be hosted on Zenodo. }
\end{itemize}

\bigskip

\begin{enumerate}[resume*=listWWNumxiv,start=1]
\item 
\textbf{\ Reusability and adoptability }}
\end{enumerate}

\bigskip


To allow others to (re)use the results, access to the datasets and models will be provided, and documentation will be

\bigskip

}
